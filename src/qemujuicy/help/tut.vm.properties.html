Virtual machine properties

A virtual machine uses some properties, which define how start and handle the VM. These are the amount of host memory or host cores are used when running the VM, as an example. In addition, the associated virtual hardware (e.g. an USB port, a network chip/card or a virtual sound driver) may be defined. The properties of the currently selected VM are shown within the tabbed pane on the right side. After creation, take a quick look over the properties of the VM on all the tabs of the tabbed pane, to ensure that they are correct.

Note: the properties of a VM cannot be changed if the VM is running. Most QEMU call parameters are constructed from these properties.

Important VM properties

Two properties are important: the VM name has to be unique and the "Architecture" will define the QEMU emulator. As an example, the Architecture (emulator) "PC x86 (64 bit)" is used to run Windows or a Linux compiled for PCs, where "ARM64 (AArch64)" will emulate an ARM processor (e.g. a Raspberry Pi).

Other VM properties

Properties showing "Best guess" use the best guess assumption of QemuJuicy, taking into account the host and guest operating system, and the emulated hardware. One can have a look to this "Best guess" on the "Advanced" tab by analyzing the QEMU parameters choosen by QemuJuicy.

Due to the fact, that a situation might call for a special treatment which cannot be handled by QemuJuicy, some properties offer the choice "None/Advanced Tab". On selection of this choice, this property will not be set. One might set another value using the "None/Advanced Tab", which has its own help chapter description.
