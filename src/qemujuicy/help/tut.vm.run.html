Run a virtual machine

Run a VM, which has an operating system (OS) installed already, by clicking the "run" button, to boot the VM from the virtual harddisk:

As an alternative for booting from the virtual harddisk, QemuJuicys "QEMU boot menu" property can be set, choosing the source for a boot directly in the VMs emulator window. A un-check of "QEMU boot menu" speeds up the boot process, not waiting for an user action.

It is necessary, that the "Architecture" property (the emulator) of the VM matches the architecture of the virtual machine OS.

If the VM virtual harddisk has no OS installed, consider using a "once-only-start", asking for an image file to boot from (see help chapter "Install an operating system"):

Live Systems might not need a harddisk, cause they run very often from an "*.iso" file without persistency or from CD/DVD. In that case, one may use the "once-only-start" button instead of the run button.
QemuJuicys remembers the directory from where the last "*.iso" file was selected.

Note: An unused virtual harddisk (qcow2 format) does not consume much disk space, regardless of the maximum disk size value.
