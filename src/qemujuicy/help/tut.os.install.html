Install an operating system

An empty virtual machine hard disk will need an operating system (OS) installed to run the VM. Very often, the operating system has been downloaded from the World Wide Web as an "*.iso" file (e.g. "ubuntu-VERSION-desktop-amd64.iso" or more special "archlinux-VERSION-x86_64.iso"). Another, less usual/older possibility is an operating system on a CD/DVD.

To install the OS, the VM can be started once from the "*.iso" file or CD/DVD, but afterwards boot always from the VMs harddisk (except a reinstall, overwriting the old system).

QemuJuicy supports this, using a "once-only-start", asking for an image file to boot from: choose an "*.iso" file within a file selection dialog ("/dev/cdrom" should also work on most Linux/Unix systems, otherwise rip the CD/DVD content to an "*.iso" file). The "once-only-start" button is:

When the installation process has been finished, operating systems either reboot automatically or simply halt. QemuJuicys "once-only-start" button sets the proper QEMU parameter if a reboot happens, booting the second time from the VMs virtual harddisk.

After the successful installation , use QemuJuicys "run" button to boot the VM from the virtual harddisk:

As an alternative for OS installation, QemuJuicys "QEMU boot menu" property can be set, choosing the source for a boot directly in the VMs emulator window. A un-check of "QEMU boot menu" afterwards speeds up the boot process, not waiting for an user action.

Note: Live Systems dont need a harddisk, cause they run very often from an "*.iso" file or CD/DVD without an installed system on the harddisk. In that case, one may use the "once-only-start" button several times.
QemuJuicys remembers the directory from where the last "*.iso" file was selected. An unused virtual harddisk (qcow2 format) does not consume much disk space, regardless of the maximum disk size value.
