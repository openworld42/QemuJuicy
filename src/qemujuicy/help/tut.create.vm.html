Create a VM

A virtual machine usually has a virtual "harddisk" (a file on the host system) and some properties, which define how start and handle the VM (e.g. how much memory or cores are used) and the associated virtual hardware (e.g. an USB port, a network chip/card or a virtual sound driver). To make this VM creation easy, just start the Vm creation wizard, a button to the right of QemuJuicy:

All of these buttons have tool tips if you stay on them, explaining their function.

The wizard will guide through the VM creation. Choose an operating system (OS) for the VM, a unique VM name or - if you like - another image icon. Set the maximum hard disk size. The real size of the virtual disk will be much less than that if the file system of the OS supports holes (common for Linux or Windows) and grows automatically with the space needed, using QEMUs "qcow2" disk format.

Choose all other properties and press the "Finish" button. A virtual machine will be created, its properties are set to specified values or defaults. In addition, the virtual harddisk will be initialized.

The new VM will show up in the VM list with its icon and its properties are shown within the tabbed pane on the right side.
